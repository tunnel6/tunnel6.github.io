# ⚡ 性能优化建议指南

> 合理配置通道数量和带宽限制，可以有效提升系统性能。监控资源使用情况，及时调整配置。

## 📊 性能监控指标

### 关键性能指标
- **CPU 使用率**: Edge 节点 CPU 占用百分比
- **内存使用率**: Edge 节点内存占用百分比
- **网络带宽**: 实时上传/下载带宽使用
- **连接数**: 当前活跃连接数量
- **延迟**: 端到端通信延迟（RTT）

### 监控工具
- YAT 控制台实时监控面板
- `top` / `htop` 系统监控
- `iftop` 网络流量监控
- `iostat` 磁盘 I/O 监控

## 🛠️ 性能优化策略

### 1. Edge 节点优化

#### 资源配置
```yaml
# config.yaml
performance:
  max_connections: 1000          # 最大连接数
  bandwidth_limit: 100mbps       # 带宽限制
  keep_alive: 30s                # 连接保活时间
  worker_threads: 4              # 工作线程数
```

#### 硬件优化
- **CPU**: 推荐 2 核以上，高并发场景推荐 4 核+
- **内存**: 推荐 2GB+，每 100 并发连接增加 512MB
- **磁盘**: SSD 硬盘，确保日志写入性能
- **网络**: 千兆网络接口，上行带宽充足

### 2. 隧道配置优化

#### 传输模式选择
| 场景 | 推荐模式 | 说明 |
|------|----------|------|
| 局域网内 | P2P Direct | 最低延迟，最高性能 |
| 移动网络 | Relay Mode | 更稳定，兼容性更好 |
| 复杂 NAT | Relay Mode | P2P 成功率低 |
| 高安全性 | Relay Mode | 所有流量经过 Edge 节点 |

#### 通道配置
- **通道数量**: 根据业务需求合理配置，避免过多通道导致资源浪费
- **带宽分配**: 为关键业务通道分配更高带宽优先级
- **超时设置**: 合理设置连接超时，避免资源占用

## 🚨 性能问题诊断

### 高 CPU 使用率
- **可能原因**: 
  - 连接数过多
  - 加密计算密集
  - 日志级别过高
- **解决方案**: 
  - 限制最大连接数
  - 升级硬件配置
  - 调整日志级别为 WARN 或 ERROR

### 高内存使用率
- **可能原因**: 
  - 缓存过大
  - 连接泄漏
  - 大文件传输
- **解决方案**: 
  - 调整缓存大小配置
  - 检查连接管理逻辑
  - 启用流式传输处理大文件
